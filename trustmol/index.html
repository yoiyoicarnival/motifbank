なぜブラックボックスは危険か

分子MLモデルは訓練域外で「自信満々に」間違える。これが知られていても解決策がなかった。

🧪

訓練域外で崩壊

QSARモデルは新規スキャフォールドに対して2-126×のエラー増幅を示す。現行の手法はこれを予測できない。

🤫

沈黙の失敗

アンサンブル法やGPは「自信あり」と判定しながら大きく間違える。信頼スコアとエラーが相関しない。

📋

規制対応ができない

FDA/EMAは「適用ドメイン」の明示を要求する。しかし数学的保証を持つ手法が存在しなかった。

どう解決するか

MotifBank Theorem 3: モデルの中身を開かずに、幾何学的距離だけで信頼性を判定する。

Theorem 3 — Hallucination Rejection (Geometric OOD)
d_min = mintrain RMSD(query_geom, train_geom)

IF d_min < ε (= 0.10 Å)
  THEN TRUST  — |error| ≤ L × ε (Lipschitz保証)

IF d_min ≥ ε
  THEN REFUSE  — 期待エラー増幅: 126×

モデル非依存

モデルの重み・アーキテクチャ不要。訓練データの幾何学バンクさえあれば判定できる。 GNN、MPNN、SchNet、AIMNetなど任意のモデルに適用可能。

規制対応グレード

Lipschitz定数Lが既知なら|error|≤L×εが数学的厳密命題。 FDA/EMAの「適用ドメイン」要件に直接対応できる唯一の手法。

API リファレンス

既存のMLパイプラインに1行追加するだけ。

# クエリ分子のOOD判定 — 実装済み POST /v1/check curl -X POST http://localhost:8000/v1/check \ -H "X-API-Key: YOUR_KEY" \ -H "Content-Type: application/json" \ -d '{ "atoms": [ {"symbol": "Si", "x": 0.0, "y": 0.0, "z": 0.0}, {"symbol": "O", "x": 1.6, "y": 0.0, "z": 0.0}, {"symbol": "O", "x": 0.0, "y": 1.6, "z": 0.0} ], "eps": 0.10, "L_geom": 0.705 }' # ✓ In-domain hit: { "trustworthy": true, "d_min": 0.031, "guarantee": "lipschitz", "error_bound_ha": 0.0219, "error_bound_kcal": 13.74, "reject_reason": null } # ✗ OOD → rejected: { "trustworthy": false, "d_min": 0.412, "guarantee": "none", "reject_reason": "OOD: d_min=0.412 ≥ ε=0.1", "expected_error_amplification": 126 }
# バッチ判定 (最大1000分子/リクエスト) curl -X POST https://api.trustmol.io/v1/batch \ -H "Authorization: Bearer YOUR_API_KEY" \ -d '{ "molecules": [ {"smiles": "CC(=O)Nc1ccc(O)cc1"}, {"smiles": "c1ccc(N)cc1"}, {"smiles": "O=C(O)CCCC(=O)O"} ], "bank_id": "drug-fda-2024" }' # Response: { "results": [...], "summary": {"trusted": 2, "rejected": 1} }
# カスタムバンクのビルド (自社訓練データ) curl -X POST https://api.trustmol.io/v1/bank/build \ -H "Authorization: Bearer YOUR_API_KEY" \ -F "smiles_file=@training_set.smi" \ -F "name=my-property-model-v2" \ -F "r_cut=5.5" # Response: { "bank_id": "usr_abc123", "n_bank": 847, "s_local": 6.74 }

Python SDK

from trustmol import TrustMol client = TrustMol(api_key="YOUR_KEY") # 単体判定 result = client.check(smiles="CC(=O)Oc1ccccc1C(=O)O", bank="drug-fda-2024") if result.trustworthy: print(f"信頼できます (誤差上界: {result.error_bound_ha:.3f} Ha)") else: print(f"OOD検知: d_min={result.d_min:.3f} Å — 予測を拒否") # 既存MLパイプラインへの統合 def safe_predict(smiles, model, bank_id): trust = client.check(smiles=smiles, bank=bank_id) if not trust.trustworthy: raise OODError(f"OOD: d_min={trust.d_min:.3f} Å") return model.predict(smiles)
